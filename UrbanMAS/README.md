# UrbanMAS - Многоагентная система моделирования градостроительной деятельности

## Описание проекта

UrbanMAS (Urban Multi-Agent Simulation) - это многоагентная система для моделирования и анализа градостроительной деятельности на языке Julia. Система разработана с применением методологии ОТСУ (Общая теория систем Урманцева) и ОТС (Общая теория систем Уёмова).

## Основные возможности

### 1. Моделирование участников строительного процесса
- **Уровень 0**: Государственные органы (Минстрой, Ростехнадзор, региональные и муниципальные власти)
- **Уровень 1**: Заказчики и инвесторы (государственные и частные застройщики)
- **Уровень 2**: Генеральные подрядчики и проектные организации
- **Уровень 3**: Субподрядчики и специализированные организации
- **Уровень 4**: Поставщики материалов и оборудования
- **Уровень 5**: Эксплуатирующие организации

### 2. Поддержка российского законодательства
- Градостроительный кодекс РФ
- Федеральный закон № 214-ФЗ (долевое строительство)
- Федеральный закон № 44-ФЗ (госзакупки)
- Федеральный закон № 223-ФЗ (закупки госкомпаний)
- СРО и лицензирование

### 3. Экономическое моделирование
- Управление портфелем инвестиционных проектов
- Расчет NPV, IRR, PI
- Анализ денежных потоков
- Оптимизация капитальных вложений
- Моделирование различных схем финансирования (бюджетное, ГЧП, частное)

### 4. Управление жизненным циклом
- Предынвестиционная фаза
- Проектирование и экспертиза
- Строительство
- Ввод в эксплуатацию
- Эксплуатация и модернизация
- Вывод из эксплуатации

### 5. Анализ рисков
- Регуляторные риски
- Финансовые риски
- Технические риски
- Риски сроков
- Метод Монте-Карло
- Анализ чувствительности

### 6. Системные метрики (ОТСУ/ОТС)
- Коэффициент целостности системы
- Уровень эмерджентности
- Индекс стабильности
- Потенциал развития
- Адаптационная способность

## Быстрый старт

```julia
using UrbanMAS

# Создание среды
env = UrbanEnvironment(
    region = Region("Московская область"),
    horizon = SimulationHorizon(2024, 2030)
)

# Создание агентов
gov_agent, gov_data = create_government_agent(
    "Минстрой России",
    Federal,
    "РФ",
    authority = [:construction_permit, :expert_review],
    budget_total = Money(1e12)
)

dev_agent, dev_data = create_developer_agent(
    "СтройИнвест",
    PrivateCompany,
    initial_assets = Money(5e9)
)

# Создание проекта
project = Project(
    id = uuid4(),
    name = "ЖК \"Северный\"",
    type = :residential,
    status = Planning,
    location = "Москва",
    total_area = 50000.0,
    estimated_cost = Money(3e9),
    funding_sources = [(:equity, Money(1e9)), (:debt, Money(2e9))],
    timeline = Dict(
        :start => Date(2024, 1, 1),
        :end => Date(2026, 12, 31)
    ),
    permits = Document[],
    contractors = UUID[],
    completion_percentage = 0.0,
    risks = RiskItem[]
)

# Добавление проекта в портфель
add_project!(dev_agent, project)

# Получение разрешения
permit = obtain_permit(dev_agent, project.id, ConstructionPermit, gov_agent.id)

# Запуск симуляции
scenario = Scenario(
    name = "Программа жилищного строительства",
    description = "Моделирование программы строительства жилья на 2024-2030 гг."
)

engine = SimulationEngine(environment = env)
results = run_simulation(engine, scenario)

# Анализ результатов
metrics = calculate_system_metrics(env)
println("Системная целостность: ", metrics.wholeness_indicator)
println("Уровень эмерджентности: ", metrics.emergence_level)
```

## Архитектурные принципы

### ОТСУ (Урманцев)
- **Иерархичность**: 6 уровней системы от государства до эксплуатантов
- **Целостность**: Коэффициенты целостности для каждого агента
- **Эмерджентность**: Отслеживание возникающих свойств системы

### ОТС (Уёмов)
- **Системный подход**: Анализ взаимосвязей между агентами
- **Параметризация**: Количественная оценка системных свойств
- **Динамика**: Моделирование эволюции системы во времени
